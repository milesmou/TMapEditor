# TMap Editor

基于 .NET 10 + Avalonia 12 的独立跨平台地图编辑器，编辑源文件使用可读 JSON 格式 `.tmap`，烘焙结果兼容现有游戏地图加载结构。

格式定义见 [TMAP_FORMAT.md](TMAP_FORMAT.md)。

## 启动

双击 `run.cmd`，或执行：

```powershell
dotnet run --project .\TMapEditor.csproj
```

## 已支持

- `.tmap` 新建、打开、保存和另存为
- 修改后在用户执行保存或“烘焙导出”时写入 `.tmap`；关闭、打开或新建时会提示处理未保存修改
- 启动时自动打开上一次成功打开或保存的工程
- 工程资源库：导入 PNG/JPG/WebP/BMP 时复制到 `Resources`，保存缩略图和相对路径
- 资源缩略图支持右键删除；正在被场景元素使用的资源会阻止删除
- 从右下角资源视图拖放图片到图片层或对象层；对象层图片作为动态元素单独导出
- 资源缩略图支持 50%–200% 预览缩放
- 图片层和对象层统一显示在图层列表中，均可新增多个、删除、重命名、调整前后层级和控制显示；列表越靠上越在前景，新工程默认无图层
- 图片选择、拖动、旋转、正负缩放、锚点和绘制顺序
- 对象层图片支持独立 `Z`，数值越大绘制越靠前并随动态图片数据导出
- 滚轮缩放，中键或空格+左键平移画布
- Walk / Block 单格状态画刷；重复绘制直接覆盖格子状态
- 独立格子 `Z` 画刷；支持连续绘制、矩形框选、清除、显示隐藏和撤销
- 可从已标记的行进格计算可达区域，将不可达格转为阻挡格，并将其余未标记格转为行进格
- 地图元素多选和批量删除
- 元素视图跟随当前选中图层，显示该图片层或对象层中的对象点与图片元素
- 画布可直接选中任意可见层级的元素，并自动切换到元素所在层级
- 工具栏的“路点”选项用于显示或隐藏画刷格子
- 工具栏的“格子 Z”选项用于显示或隐藏非零 Z 格子及其数值
- Object 对象点放置、移动、备注、显示颜色和 `Z` 自定义；备注仅在编辑器元素列表中显示，不会导出
- 网格、Chunk 边界预览及网格吸附
- 网格、Chunk、路点、格子 Z 和吸附网格的显示设置随工程保存并在打开时恢复
- 导出 `chunk_row_col.png`、包含层级数据的 `Grid.json` 和独立路点文件 `GridPath.json`
- 导出前校验全部图片引用，并清理本工具上次生成、当前已失效的 Chunk 和图层产物
- Windows 界面固定使用 Direct3D 11（ANGLE），macOS 使用 Metal；窗口标题会显示当前界面渲染后端
- 烘焙导出优先使用 GPU，GPU 不可用或 Chunk 超出 GPU 纹理限制时自动回退到 CPU

## 操作

- 点击右下角“导入图片”把素材复制到工程的 `Resources` 目录；选择目标图层后，将资源缩略图拖到画布即可创建图片元素。图片层元素会烘焙进 Chunk，对象层元素会复制到 `<对象层名称>/images/`，并在 `Grid.json` 的 `File` 字段中记录不带扩展名的文件名。
- 场景图片只能从资源视图拖入，不支持绕过资源目录直接导入到图层。
- 拖入图片前先新增并选择目标图层；烘焙目录以及 `Grid.json` 中的图片层字段使用定义的层级名称。
- 选择“行进区域画刷”或“阻挡区域画刷”：按住鼠标左键沿路径连续刷格子，按住鼠标右键拖动则框选一个矩形范围，松开后统一刷入；`Esc` 可中断当前操作。
- “编辑 → 优化阻挡区域”以上下左右相邻的行进格作为可达范围，将无法从任何已标记行进格到达的格子填为阻挡格，其余未标记格填为行进格；操作支持撤销。
- 选择“清除格子画刷”可用相同的左键连续操作或右键框选操作把格子恢复为空状态。
- 在工具栏输入非零整数并点击“刷 Z”，可用左键连续绘制或右键框选格子 Z；“清除 Z”只移除格子 Z，不改变 Walk / Block 状态。未设置格子的 Z 默认为 `0`。
- 在地图元素列表中使用 `Ctrl` / `Shift` 多选，或在画布中使用 `Ctrl + 单击` 多选；按 `Delete` 或右键选择“删除”可批量删除。
- 选中一个或多个地图图片元素后，可用 `W/A/S/D` 每次微调 1 个地图单位的位置。
- 点击左侧图片层或对象层，“元素视图”会直接显示该层中的元素；行进和阻挡格不会显示在元素列表中。
- 在画布中点击任意可见且未锁定的元素，会自动切换到该元素所在层级，不受当前层级限制。
- 地图元素和对象元素列表末尾的锁形按钮可锁定元素；锁定后无法从画布中选中，仍可从列表中解锁。
- 选中目标对象层后，在地图空白处单击即可放置对象点；拖动已有对象点可移动。
- “烘焙导出”会先保存当前 `.tmap`，再选择游戏地图输出目录，例如 `assets/bundles/maps/map1/map1`；未命名的地图会先弹出保存对话框，取消或保存失败时不会继续导出。
- “烘焙导出”会记住上一次选择的输出目录，并在下次导出时默认打开该目录。

## 命令行烘焙导出

无需打开编辑器窗口即可导出 `.tmap`：

```powershell
.\TMapEditor.exe --export "D:\Maps\map1.tmap" --output "D:\Game\assets\bundles\maps\map1\map1"
```

成功时进程退出码为 `0`，参数错误为 `2`，读取或导出失败为 `1`。使用 `dotnet TMapEditor.dll` 执行相同参数时可直接看到导出结果或错误信息。

导出完成信息末尾会显示本次烘焙实际使用 `GPU` 还是 `CPU`。命令行模式同样会优先创建 GPU 上下文，创建失败时自动使用 CPU。

图片路径在保存时相对于 `.tmap` 文件保存。建议把 `.tmap` 和素材目录放在同一个地图工程目录中，便于整体移动。
